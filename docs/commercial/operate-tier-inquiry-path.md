# Operate Tier Inquiry Path

Issue: hummbl-dev/hummbl-dev#76

Status: inquiry-only draft. Do not add checkout pressure or automatic acceptance.

## Purpose

`Operate` is for ongoing help with agentic work governance, review workflows, release risk, receipts, and public/private routing. It should remain inquiry-only until scope, capacity, delivery responsibility, data handling, and support boundaries are reviewed by a human.

## Buyer Fit

Good fit:

- founder-led technical teams adopting AI agents
- teams with active repo, PR, or release workflows
- teams needing review gates and receipts
- teams that can define a bounded operating cadence

Poor fit:

- requests for legal, compliance, security, medical, or audit certification
- emergency incident response
- broad "run everything" requests
- work requiring production secrets before trust and scope are established
- unclear ownership or no review path

## Inquiry Questions

Ask:

- What agentic workflow do you want help operating?
- Which repos, teams, or public surfaces are in scope?
- What decisions should remain human-only?
- What review gates already exist?
- What receipts or audit evidence do you need?
- What cadence are you considering?
- What data or access would be required?
- What must stay private?

Do not ask by default for secrets, customer data, production access, or regulated information.

## Human Review Gates

Human approval is required before:

- accepting recurring responsibility
- requesting production access
- touching customer data
- making delivery, refund, or SLA promises
- representing legal, compliance, security, medical, or audit conclusions
- naming private customers or private repos in public material

## Public-Safe Response

```text
Operate is inquiry-only. We can review the workflow, boundaries, cadence, and evidence needs before deciding whether ongoing support is appropriate. Please send a sanitized description first; do not send secrets, customer records, regulated data, or production credentials.
```

## Capacity Notes

Track privately before acceptance:

- weekly time budget
- response expectations
- handoff owner
- escalation path
- data boundaries
- stop conditions
- renewal or closeout criteria
