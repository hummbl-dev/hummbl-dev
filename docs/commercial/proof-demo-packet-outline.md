# Proof / Demo Packet Outline

Issue: hummbl-dev/hummbl-dev#76

Status: public-safe outline.

## Purpose

Give buyers enough evidence to understand HUMMBL's agent-governance work without exposing private customer work, private repos, secrets, or unsupported claims.

## Packet Sections

1. **Problem**
   - agentic work needs authority, review, receipts, rollback, and source discipline
   - public examples only

2. **Approach**
   - runtime governance primitives
   - issue-first execution
   - non-author review gates
   - source and receipt requirements

3. **Public examples**
   - public docs, PRs, or packages only
   - no private customer or household material

4. **Demo path**
   - choose a public repo or sanitized sample workflow
   - show issue -> branch -> PR -> review -> receipt
   - explain stop conditions

5. **What this proves**
   - reviewability
   - provenance
   - bounded delegation
   - public/private routing discipline

6. **What this does not prove**
   - legal, compliance, security, medical, or audit certification
   - guaranteed customer outcomes
   - production readiness without scoped implementation
   - private-work claims not visible in public evidence

## Allowed Evidence

- public PRs and merged commits
- public package links
- public docs and runbooks
- sanitized screenshots without private data
- explicit receipts from public issues

## Disallowed Evidence

- customer records
- private repo contents
- credentials or configuration secrets
- payment or webhook internals
- personal health, household, relationship, job-search, or legal-outreach details
- unreviewed claims presented as proof

## Review Checklist

- [ ] Every claim links to a public source or receipt.
- [ ] Private context is summarized generically or omitted.
- [ ] No regulated, customer, credential, or personal data appears.
- [ ] Claims are framed as evidence, not certification.
- [ ] Known limits are explicit.
