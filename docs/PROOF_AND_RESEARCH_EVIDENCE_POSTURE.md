# Proof And Research Evidence Posture

This repository contains multiple kinds of public proof assets. They are useful, but they do not all carry the same evidentiary weight.

## Evidence Tiers In This Repo

### 1. Publicly verifiable repo/package evidence

Strongest surface for external reuse:

- package metadata
- public README claims tied to current repo state
- versioned files in public repos
- publicly visible issue/PR/release history

Examples:

- `dependencies = []` in a public `pyproject.toml`
- model counts documented in a current public repo
- package presence on PyPI or npm

### 2. Primary-source-backed research synthesis

Useful for positioning, briefing, and internal/external strategy work, but still requires care:

- research round summaries
- incident syntheses
- market comparison notes
- regulatory or liability briefs

These artifacts may accurately summarize source material, but they are still synthesized documents. Before high-stakes external use, re-check the cited primary material.

### 3. Narrative positioning and case-study interpretation

Useful for explaining why the work matters:

- case studies
- one-pagers
- capability briefs
- pitch-oriented summary language

These should distinguish clearly between:

- what a public artifact proves directly
- what is interpretation or architectural synthesis
- what is a dated snapshot that may drift

## Rules For External Reuse

- Treat counts, scores, and test totals as **snapshot claims** unless tied to a date.
- Treat research corpus summaries as **source-backed synthesis**, not self-authenticating canon.
- Treat case studies as **public proof surfaces plus interpretation**, not sworn operational records.
- Re-check legal, regulatory, insurance, and compliance claims against primary sources before publishing or using them in contracts, diligence, or formal advice.
- Do not describe any research artifact here as legal advice.

## Practical Shortcut

Before reusing a claim externally, ask:

1. Is this directly visible in a current public repo or package surface?
2. Is this a synthesis of external sources rather than a primary artifact?
3. Is the number/date/version still current?
4. Would a skeptical buyer, reviewer, or lawyer ask for the primary source?

If the answer to 2, 3, or 4 is yes, re-verify before reuse.
