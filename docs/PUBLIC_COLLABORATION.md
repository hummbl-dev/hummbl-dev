# Public Collaboration

This repository routes public profile, repo-discovery, and cross-repo collaboration for the `hummbl-dev` account.

## Where to Start

| Need | Route |
|---|---|
| Bug in a specific repo | Open an issue in the target repo. |
| Bug in this public profile, tool index, or static page | Open a bug report in this repo. |
| Factual correction, stale claim, or broken source | Open a content correction in this repo. |
| Cross-repo proposal | Open a cross-repo proposal in this repo. |
| Agent handoff or bounded task | Open an agent handoff issue in this repo or the target repo. |
| Security concern | Use `SECURITY.md`; do not disclose vulnerabilities in public issues. |

## Boundaries

Public issues should not include secrets, credentials, private operational receipts, client-sensitive details, or private repo content. If a proposal depends on private context, describe the public-facing outcome and keep sensitive evidence out of the issue.

## Review Expectations

Keep proposals specific enough to review:

- State the target repo or public surface.
- Explain the expected user or maintainer benefit.
- Link public evidence when making factual claims.
- Separate adopted behavior from exploratory ideas.
