# Repo Maturity Model

Public HUMMBL repos use lightweight maturity tags so visitors can tell what is ready to use, what is exploratory, and what is retained for reference.

| Status | Meaning |
|---|---|
| `seed` | Early public seed with scope, README, license, and initial issues. Expect change. |
| `exploratory` | Active investigation or prototype. Useful, but not stable guidance. |
| `active` | Maintained project with current work, issue triage, and review expectations. |
| `stable` | Mature surface with documented behavior and low expected churn. |
| `maintenance` | Usable but mostly receiving fixes, docs updates, or dependency care. |
| `archived` | Retained for reference. Do not treat as current unless explicitly marked. |
| `private/not-public` | Known org surface that is not part of the public collaboration path. |

## Use

Status tags describe repository maturity, not legal, compliance, or security assurance. A `seed` or `exploratory` repo can be useful without being canon. A `stable` repo still needs source-specific review before operational adoption.
